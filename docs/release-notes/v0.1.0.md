# Evanesca v0.1.0 Release Notes

Status: draft release notes for the first public open-source release.

## Release Theme

This release reorganizes Evanesca as an open-source DeFi forensic reconstruction
and measurement framework. It is focused on source availability, public
documentation, a technical report, reproducible examples, and a stable JSON
contract for future evidence exploration.

Public repository:

```text
git@github.com:appff/Evanesca_public.git
```

This release is not positioned as a production exploit detector.

## Included

- Public README and project positioning.
- Technical report draft under `docs/technical-report/`.
- Public analysis-result JSON schema under `docs/public-api/`.
- Experimental transaction analysis export command:

```bash
npm run analyze -- --tx <transaction-hash> --chain ethereum --out out/analysis.json --pretty
```

- Synthetic schema example under `docs/example-datasets/`.
- Replayed bZx case-study JSON under `docs/example-datasets/`.
- LaTeX/PDF technical report under `docs/technical-report/`.
- Build-status audit under `docs/source-code/`.
- Example dataset manifest and validator:

```bash
npm run validate:public-artifacts
```

- Reproducibility guide and artifact planning documents.
- Public artifact manifest with count validation for the current defect and
  non-baseline hash bundles.
- Contribution, security, citation, dependency-audit, and release-status files.
- Public repository export helper:

```bash
bash scripts/export-public-repo.sh
```

- GitHub Actions smoke test for public repository checks.

## Known Limitations

- Public transaction examples still need to be regenerated from curated
  replayed transactions.
- Constraint descriptions in the technical report still need to be synchronized
  with implementation predicates.
- Existing research artifacts require provenance and disclosure review before
  being treated as public release artifacts.
- Historical paper-era files under `docs/papers/evanesca/` still need an
- Historical paper-era files under `docs/papers/evanesca/` are excluded from
  the public repository export.
- Dependency audit findings remain a release blocker.
- Default `npm install` may fail on Puppeteer browser download/cache setup;
  use `PUPPETEER_SKIP_DOWNLOAD=true npm install` for the current CLI/artifact
  workflow.

## Frontend

Frontend work is intentionally deferred until the technical report, public JSON
schema, and at least one curated replayed transaction example are stable.
